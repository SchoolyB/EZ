4. EZ is interpreted for now, but the long-term goal is to become a compiled language.

5. EZ must be able to run on any OS or machine in the future.

6. EZ must be extremely easy for anyone to learn. A beginner should be able to pick 
   it up with minimal friction.

7. EZ has a low barrier to entry and a high skill ceiling.
   Every design decision MUST be evaluated by two core questions:
      A. Does this make it easier for a beginner to start?
      B. Does this give an expert more control or expressiveness WITHOUT confusing the beginner path?

8. EZ should provide the programmer with all the essential tools they need 
   WITHOUT requiring custom libraries just to accomplish basic tasks. 
   The standard library should be minimal enough to teach real fundamentals but expansive enough to provided the neccessary tools for advanced use.
